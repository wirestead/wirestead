65 // Transmission
66 //
67 // Strategy-aware API (recommended):
68 //
69 // send_to() / broadcast()
70 // Behaviour depends on the configured backpressure strategy:
71 // BestEffort — non-blocking; drops data when the target client's queue is full.
72 // Reliable — blocks the calling thread until queue pressure is relieved,
73 // then enqueues for send_to().
74 // broadcast() always performs non-blocking fan-out to connected clients.
75 // It does not wait for one slow client to relieve backpressure.
76 //
77 // Explicit API (escape hatch):
78 //
79 // try_send_to() / try_broadcast()
80 // Always non-blocking and always drops on full queue, regardless of strategy.
81 // Rejects payloads that would exceed the current non-blocking queue
82 // threshold; payloads below MAX_BUFFER_SIZE can still be rejected when
83 // they exceed the current backpressure high-water budget.
84 //
85 // send_to_blocking()
86 // Always blocks until queue pressure is relieved, regardless of strategy.
87 // This call may block indefinitely while the server remains active and
88 // backpressure does not clear. stop() from another thread is expected to
89 // unblock waiting senders.
