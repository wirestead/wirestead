77 // Transmission
78 //
79 // Strategy-aware API (recommended):
80 //
81 // send() / send_line()
82 // Behaviour depends on the configured backpressure strategy:
83 // BestEffort — non-blocking; drops data when the send queue is full.
84 // Reliable — blocks the calling thread until queue pressure is relieved,
85 // then enqueues. Never drops due to backpressure alone.
86 //
87 // Explicit API (escape hatch):
88 //
89 // try_send() / try_send_line()
90 // Always non-blocking and always rejects a full/backpressured queue,
91 // regardless of strategy. Reliable channels do not enqueue into pending_
92 // for try_send().
93 // Rejects payloads that would exceed the current non-blocking queue
94 // threshold; payloads below MAX_BUFFER_SIZE can still be rejected when
95 // they exceed the current backpressure high-water budget.
96 // Use when you need BestEffort behaviour on a Reliable channel for a specific
97 // call (e.g. fire-and-forget heartbeat).
98 //
99 // send_blocking() / send_line_blocking()
100 // Always blocks until queue pressure is relieved, regardless of strategy.
101 // This call may block indefinitely while the channel remains active and
102 // backpressure does not clear. stop() from another thread is expected to
103 // unblock waiting senders.
